# Architecture Decision Records

Short records of the decisions that shape DM-Instamap, so the rationale is not
lost over time. Each ADR is a numbered Markdown file with a status (Accepted,
Superseded, Deprecated), the context, the decision, and its consequences.

When a decision changes, add a new ADR that supersedes the old one instead of
rewriting history. Keep them short: a screen or less.
